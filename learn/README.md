# learn/（仓库研究笔记）

说明：
- `learn/` 是本仓库的研究笔记目录（中文为主；保留必要的英文术语/标识符）。
- 链接尽量使用仓库相对路径，保证本地与远端一致可用。

快速入口：
- 索引：`SUMMARY.md`
- 会话日志：`sessions/`
- 兜底主题：`misc/`

日常维护流程（推荐）：
1. 在当天 `sessions/YYYY-MM-DD.md` 写本次目标/问题列表。
2. 将结论沉淀为 1–3 篇主题笔记（放到对应 topic 目录或 `misc/`）。
3. 更新 `SUMMARY.md`：补充笔记链接 + 添加当天 session 链接（倒序）。
4. 回到 session 日志的“产出”里补上本次笔记链接。

目录结构：
- `sessions/`：按日期的讨论记录（先记录，再整理成主题笔记）。
- `misc/`：暂时无法归类/跨模块内容（后续可移动）。
- `<topic>/`：按 topic 划分的研究笔记目录（只允许单层）。
  - 目录深度约束：只允许 `learn/<topic>/`，不要创建 `learn/a/b/`。
  - 如果 topic 不够细：新增一个新的 topic 目录（仍保持单层）。

## 主题笔记模板（复制后修改）

```md
# <topic>：<要回答的问题>

日期：YYYY-MM-DD
主题：<topic>（learn/<topic>/）
相关代码：
- path/to/file.ext:line（symbol） - 说明
结论一句话：<一句话先给答案>

## 问题
- <我们要搞清楚什么?>

## 结论
- <结论要点 1>
- <结论要点 2>

## 依据（代码与文档线索）
- `path/to/file.ext:line`：<相关类型/函数/常量>，<为什么能支持结论>

## 反例/边界情况
- <什么时候不成立/有哪些例外/哪些点未验证>

## 相关链接
- [会话：YYYY-MM-DD](../sessions/YYYY-MM-DD.md)

## 后续问题
- <下一步要研究什么>
```

## 会话日志模板（复制后修改）

```md
# 会话：YYYY-MM-DD

## 目标
- <本次要解决的 1–3 个问题>

## 讨论记录
- <关键结论/线索（可粗略）>

## 产出
- <链接到本次新增/更新的主题笔记>

## 待办
- <下次要做的事>
```
